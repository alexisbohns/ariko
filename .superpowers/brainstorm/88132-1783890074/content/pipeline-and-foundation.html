Which foundation runs it? (media is the deciding factor)

Click the one that feels right — or tell me it's wrong and why.

A

Git-as-database + references

Metadata = YAML/JSON in a repo. Media = mostly URLs to where it already lives (SoundCloud, Figma, apps); orphan files go in Git LFS or a cheap bucket. Agents & Actions commit entries; site rebuilds from files. Version history is your audit trail — very on-theme.

Cost: cheapest to run, no server. Pain: phone/2am capture is clunky; LFS for real media gets ugly; no live querying.

B

Real backend + DB + object storage

atome becomes an app you log into. Metadata in a DB (Mongo/Postgres), media in an object store (S3 / R2 / Supabase). Capture from anywhere, query anything, real admin UI.

Cost: biggest build, something to host & maintain. Win: no ceilings — most future-proof for a "massive portfolio".

C

Hybrid — DB vault + object storage, static public site ★ recommended

A lightweight DB + object store hold the private vault and power a small admin (triage, capture, upload media). Publishing exports the chosen subset to the static portfolio you already built. Messy private side and clean public side stay separate.

Why this: handles media properly, gives you capture-anywhere, keeps the public site fast & simple, and each source can be wired in one at a time.
